Quick Help to Chiroptera Client

Keyboard shortcuts

PageUp / PageDown - Scroll the screen one page up/down

Ctrl-ArrowUp / Ctrl-ArrowDown - Scroll the screen one line up/down

ArrowUp / ArrowDown - Previous / next line in command line history

Shift-ArrowUp / Shift-ArrowDown - Search command line history using the text in the input line

LeftMouse + Drag - Select text on the screen and copy it to clipboard

RightMouse - Paste text from clipboard

Built-in Commands

/help [command] - display all commands, or help on the given command

/quit - quit the client

/eval <python code> - evaluate python code

/run <script> - run script file

/py - enter/exit python mode

Commands implemented in python modules

/list - list all triggers (triggers.py)

/trigger- define a new trigger (triggers.py)

/delete - delete a trigger (triggers.py)

/hilite - hilite a pattern (hilite.py)

/bind - bind a key to an action (bind.py)


Use /help and /help <command> to learn more about commands.

Scripting

The script engine is IronPython (http://www.codeplex.com/IronPython) which is a python implementation on .NET.

Modules

Modules, scripts that are not meant to be ran as such, but are imported and used in other scripts, reside in lib directory. The modules are:

  • batclient.py - helper functions for batclient
  • triggers.py - python functions for triggers

Scripts

Scripts that come with Chiroptera reside in std directory and user made scripts reside in <user's documents dir>/Chiroptera directory. They can be ran, for example, with /run command.

Command scripts

Script files with .bc file extension are Chiroptera command scripts. When you run one of these files, each line is executed as if you had given a command in the client.

When the client starts it runs init_std.bc script from std directory, which contains standard client initialization stuff. After that the client runs init.bc from <user's documents>/Chiroptera directory, and that's where you can put your initialization stuff.

Substitutions

Triggers and key bindings can contain substitution arguments.

  • %{expression} - expression is a python expression which is evaluated and the result is substituted in place of %{...}. For example %{"kissa" + "kala"}
  • %%{statements} - statements is a list of python statements. The statements are executed and the returned value is substituted in place of %%{...}. For example %%{i = 1; i = i + 2; return i;}
  • %<n> - n'th matched regexp group, or the whole match if n is zero. n can be from 0 to 9.
  • %L or %R - The rest of the line left of the match, or right of the match.
